Hoisting

"Hoisting is a JavaScript mechanism where variables and function declarations are moved to the top of their scope before code execution."
"For let and const : The variables are created when their containing Lexical Environment is instantiated but may not be accessed in any way until the variable's Lexical Binding is evaluated."

Bij hoisting worden de declaraties van functies en variabelen eerst gecompileerd alvorens de browser de code van het script uitvoert. Na de compilatie fase zijn de variabelen en functies gekend.

In het volgende voorbeeld wordt de functie gebruikt vóór de declaratie

sayHi('Bob'); //alert: Hi, my name is Bob

function sayHi(name) {
   alert(`Hi, my name is ${name}`);
}

sayHi('Bob'); //alert: Hi, my name is Bob

JavaScript zal enkel declaraties hoisten, geen initialisatie waarden.
De reden is dat het compilatie proces juist voor het interpreter proces komt. Het compilatie proces detecteert alle declaraties en bijhorende scope. 

console.log(x); // undefined
x = 10;
var x;
console.log(x); // 10

let en const: Hoisting gebeurt ook voor let en const, maar je mag er maar naar refereren als de "actual declaration" geëvalueerd is at runtime (ES6 specificatie) = de Temporal Dead Zone

x = 10; // Uncaught ReferenceError: x is not defined
let x;
console.log(x); //10
